- **v2.23.0 (derive marker→value polarity from configurable values map (fix emit_configurable_alert inversion); closes #775):** `contract.lib.runtime._resolve_marker_value` previously HARDCODED the legacy human-approval marker polarity (marker PRESENT → value `"false"`, ABSENT → `"true"`).
